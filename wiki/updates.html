Wiki Updates - March 2018


Campaign Automation

  • The default operator for a CA decision node is and. In order to avoid a participant getting stuck in a decision node, customers should specify which operator they want to use.
  • Leads and contacts going through the same node have to be addressed in separate condition statements in order for a participant to process correctly.
  • Emails sent from a Campaign Automation's Send Email action can add Google Analytics UTM parameters. By default, a created Emails record will have the Add Google Analytics field set to yes, so the Send Email action simply needs to include a CRM campaign in order for the UTM parameters to be created and added to the email links.
  • Dev has added the steps our plugin takes when to work with an Added to List trigger.
  • When one Email Interaction trigger is checking multiple Emails for clicks/opens, by default the trigger will use and logic. If a customer wants to use or logic instead, they will have to set up parallel triggers to check the Emails individually for opens/clicks.
